How to use

LinkedIn Job Parser — public data only

What this is

Scrapes public LinkedIn job postings (no login, no API key). Results are saved to a local database (deduped). Intended for personal, low-volume use only — please respect LinkedIn's Terms of Service.

Tabs

  • Jobs — search LinkedIn and save listings. Each search appends new results; duplicates are skipped.
  • Companies — run a search and see which companies are hiring for that role, grouped by employer. Use Show saved to browse without a new scrape.
  • Explore — browse all your saved positions and companies without hitting LinkedIn. Click a position to see who's hiring. Use Search more to fetch fresh results for any saved position.

Search tips

  • Keywords — job title, skills, or role (e.g. "python developer", "data scientist").
  • Location — city, country, or "Remote". LinkedIn is fuzzy here; leave blank for worldwide.
  • Workplace — filter to Remote, Hybrid, or On-site only.
  • Date posted — limit to jobs posted in the last 24 h / week / month.
  • Max results — up to 200 (each page is 25 listings, so 200 = 8 pages).
  • Fetch full details — fetches each job's detail page for seniority, employment type, full description, and applicant count. Slower (one extra request per listing) but richer data.

Positions & normalization

Searches are grouped into positions by keyword + location. Common seniority prefixes (Senior, Junior, Lead, etc.) are stripped, so "Senior Python Developer" and "Python Developer" land in the same position group.

Limitations

  • Rate limiting — LinkedIn blocks rapid requests. Searches include jittered delays (2–5 s per page) and automatic 429 backoff. A large search can take a minute.
  • People search (Find leaders) — LinkedIn's public people search is login-gated. The button attempts a best-effort scrape but usually returns no results.
  • Fragile selectors — LinkedIn changes its HTML markup without notice. If results are empty or fields are missing, selectors in src/parser.py may need updating.
  • Public data only — no private profiles, no messages, no company internal data.
